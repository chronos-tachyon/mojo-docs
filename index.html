About Mojo

View the source at github.com/chronos-tachyon/mojo. Offered under the MIT License.

Mojo is a toolkit for C++ Bazel users. The long-term goal is for Mojo to be for C++ what the Python or Go standard libraries are for their respective languages. The immediate focus for early versions of Mojo is to build the infrastructure for network servers powered by asynchronous event-driven dispatch. This will eventually include a basic but standards compliant HTTP/1.1 and HTTP/2.0 web server library.

While Mojo is original code, it was not written in a vacuum. It takes inspiration from a number of sources:

  • The C++ codebase at a previous employer
  • The Go standard library
  • The Python standard library

Exported features fall into one of the following categories:

  • FROZEN: APIs are frozen. No changes except bug fixes.
  • STABLE: Existing APIs are stable, but new APIs may be added.
  • RC: APIs are provisional and subject to change.
  • BETA: APIs are in flux.

Packages

base

The base package is a toolkit of helpers for various small tasks: logging, building message strings, returning standardized error codes, measuring and manipulating time, and several others.

event

The event package, loosely inspired by the Python 3 asyncio module, provides event loops, thread pools, and other tools for asynchronous and parallel processing. Most significantly, the package provides event::Manager, a class for asynchronously handling events from file descriptors, signals, timers, and other sources.

io

The io package, which takes inspiration from the Go io package, provides a standardized API for arbitrary input and output byte streams: io::Reader and io::Writer.

net

The net package, which takes inspiration from the Go net package, provides networking abstractions like net::Addr and net::Conn.

path

The path package borrows elements from the Go path and Python os.path packages. It offers functions for cleaning, resolving, and manipulating pathnames.
